# 功能设计说明书 — 质量自检 Checklist

> 使用说明：Phase 4 Gate 前，SE 必须对每份功能设计说明书逐项自检。全部通过后才可展示给用户审阅。

---

## 需求点厘清

- [ ] 输入/输出/前置条件/后置条件/约束条件全部填充
- [ ] 验收标准可测试、可量化（如"响应时间≤200ms"而非"功能正常"）
- [ ] 约束条件 ≥3 条，每条标注影响范围和违反后果

## 架构决策（ADR）

- [ ] ADR 编号格式正确：`ADR-[SR编号]-[序号]`
- [ ] 候选方案 ≥2 个，每个有明确的优劣分析
- [ ] 量化对比矩阵完整（评估维度、权重、评分、加权总分）
- [ ] 选择理由具体，说明了"业界常见做法"和"本项目差异点"
- [ ] ADR 状态明确标注（提议/已决定/已替代）

## 4+1 视图

- [ ] 每个视图已逐项评估"使用/跳过+理由"
- [ ] 选用的视图在交互阶段使用 ASCII 字符图
- [ ] 未默认全部跳过，有合理的视图选择依据

## 接口契约

- [ ] 包含数据格式/字段定义
- [ ] 包含错误码/异常处理
- [ ] 包含时序约束/超时策略
- [ ] 包含幂等/并发约定
- [ ] 包含版本兼容策略
- [ ] 接口交互时序图已生成（多接口场景）

## DFX 设计要点

- [ ] 每个 DFX 维度包含：量化指标+阈值、业界基准对比、失效场景+影响等级、设计对策+验证方式
- [ ] 禁止只写"需关注"或"重要"就结束
- [ ] 引用具体行业标准号

## SR→AR 分配

- [ ] 分配编号格式正确：`AR-[IR编号]-[SR序号]-[子序号]`
- [ ] 分配依据覆盖：功能耦合度、性能瓶颈点、团队边界、部署约束、演进兼容性
- [ ] 每个 SR 至少分配 1 个 AR，零遗漏

## FMEA 分析

- [ ] 每行含：失效模式+触发条件+影响等级+检测方法+补偿措施
- [ ] S/O/D 评分完整（无空值）
- [ ] RPN = S×O×D 已计算
- [ ] 高 RPN（>100）有额外补偿措施说明

## 依赖关系

- [ ] 模块依赖关系图已生成
- [ ] 周边依赖关系已列出（网络/SDK/平台依赖）

## 格式与一致性

- [ ] 无 `[待补充]`/`[TBD]`/`[...]` 等占位符
- [ ] 编号/命名格式全局统一
- [ ] 术语与 references/ 定义一致
- [ ] 聚焦架构设计，未陷入代码实现细节
- [ ] 文档末尾附本 Checklist 自检结果
