# 需求分析说明书 — 质量自检 Checklist

> 使用说明：Phase 3 Gate 前，SE 必须逐项自检。全部通过后才可展示给用户审阅。

---

## 结构化 IR

- [ ] IR 标识编号格式正确：`IR-[版本]-[序号]`
- [ ] 5W2H 全部维度已填充（跳过的维度有明确理由）
- [ ] How 方向声明已产出（选定方向+核心策略+关键约束）
- [ ] How much 资源估算已收集

## 系统上下文

- [ ] 系统上下文图已生成（交互阶段为 ASCII，归档阶段为 PlantUML）
- [ ] 所有外部系统/角色已识别并标注交互方式
- [ ] 系统边界清晰，无歧义

## SR 分解

- [ ] SR 编号连续且格式正确：`SR-[IR编号]-[序号]`
- [ ] 每个 SR 满足 INVEST 原则（Independent, Negotiable, Valuable, Estimable, Small, Testable）
- [ ] 每个 SR 有明确的可量化验收标准（禁止"功能正常"等模糊表述）
- [ ] IR→SR 追踪矩阵完整，零遗漏（每个 IR 至少分解出 1 个 SR）
- [ ] SR 总数合理（>10 个时考虑分组，<3 个时考虑是否分解不足）

## 功能性需求

- [ ] 用例图已生成，覆盖所有核心场景
- [ ] 每个用例有明确的参与者、前置条件、后置条件
- [ ] 异常场景已识别（至少覆盖 1 个异常用例）

## DFX 非功能性需求

- [ ] 全部 DFX 维度已逐一评估（13 个维度，含"不适用"的也要写明理由）
- [ ] 每个 DFX 维度包含：量化指标+阈值、业界基准对比、失效场景、设计对策
- [ ] 引用了具体行业标准号（如 ISO 26262、IEC 62443）而非"参考行业标准"

## 架构决策

- [ ] 关键决策有 ADR 记录（编号格式 `ADR-[SR编号]-[序号]`）
- [ ] 每个 ADR 有 ≥2 个候选方案 + 量化对比矩阵

## 格式与一致性

- [ ] 文档内编号/命名格式全局统一
- [ ] 术语使用与 references/ 定义一致，未自创术语
- [ ] 无 `[待补充]`/`[TBD]`/`[...]` 等占位符
- [ ] 每个主要章节有 1-2 句摘要引导
- [ ] 文档末尾附本 Checklist 自检结果

## 系统影响分析

- [ ] 覆盖上下游系统、数据变更、接口变更、配置变更、运维变更、迁移兼容
- [ ] 不仅写了"影响模块A"，而是展开了具体影响内容
