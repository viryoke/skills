# Self-Learning — 通用自学系统

## 设计理念

**问题：** 直接学习细节会导致迷失，不知道当前知识在整个体系中的位置。

**解决方案：** 
```
科普介绍（感性认知："这个领域是怎么回事"）→ 知识图谱（理性框架："知识点如何关联"）→ 各阶段学习
```

**为什么有效：** 
- 科普用故事串联知识点，让你理解"它们为什么存在、如何自然衔接"
- 图谱展示依赖关系，让你知道"学 A 之前必须先学 B"
- 两者配合，你始终有全局视角，学习不会跑偏

## 命令

| 命令 | 作用 |
|------|------|
| `/learn <topic>` | 启动新学习任务 |
| `/learn` | 恢复未完成的任务 |
| `/learn status` | 显示进度 |
| `/learn stop` | 中途退出，保存进度 |
| `/learn stages <N>` | 调整阶段数 |

## 核心流程

### 启动任务

1. 生成 slug（topic 转小写，空格替换为 `-`）
2. 创建目录：`<slug>/` 包含 `roadmap.md`、`overview.md`、`knowledge-graph.md`、`progress.json`、`stage-1-6/`
3. 进入阶段 0

### 阶段 0：全局认知建立

**目标：** 在深入学习前，建立对领域的完整认知框架。

**为什么重要：** 有了全局认知，后续学习时你才能理解"这个知识点是什么、为什么重要、与其他知识如何关联"。

**产出三份文档：**

| 文档 | 作用 | 读取时机 |
|------|------|----------|
| `roadmap.md` | 规划：学什么、学多久 | 规划阶段 |
| `overview.md` | 科普：故事串联知识点 | 学习前 |
| `knowledge-graph.md` | 图谱：知识依赖关系 | 学习中、复习时 |

**生成流程：**
1. 搜索领域的标准学习路径、官方文档、经典案例
2. 生成三份文档（详细模板见 `references/templates.md`）
3. 闸门展示，等待用户确认

**闸门交互：**
- `O` — 查看科普介绍
- `V` — 查看知识图谱  
- `Q: <问题>` — 提出疑问
- `F: <反馈>` — 要求调整
- `N` — 确认，进入阶段 1

**成功标准：** 用户理解了领域的整体框架，能回答"这个领域是什么、核心概念有哪些、学习路径是什么"。

---

### 阶段 1-6：深入学习

每个阶段遵循相同流程，产出：`notes.md`（笔记）、`quiz.md`（测验）、`practice.md`（实战）。

**阶段目标：**

| 阶段 | 目标 | 成功标准 |
|------|------|----------|
| 1 基础 | 掌握核心概念 | 能解释术语、画出概念关系图 |
| 2 原理 | 理解底层机制 | 能回答"为什么这样工作" |
| 3 进阶 | 掌握高级主题 | 能处理复杂场景、做出技术选择 |
| 4 实战 | 完成实际项目 | 能从零搭建项目、处理真实问题 |
| 5 融会贯通 | 形成知识体系 | 能串联知识、给他人讲解 |
| 6 前沿 | 了解发展趋势 | 能识别新技术价值、规划持续学习 |

**执行流程：**

1. **研究：** 搜索高质量来源（优先官方文档），提取内容
2. **生成：** 按模板生成笔记、测验、实战（模板见 `references/templates.md`）
3. **质量检查：** 笔记有示例？测验有解析？实战可执行？
4. **闸门：** 展示摘要，等待确认
5. **完成：** 更新知识图谱，进入下一阶段

**闸门交互：**
- `Q: <问题>` — 提出疑问
- `F: <反馈>` — 调整内容（方向/深度/范围/回归路线图）
- `N` — 确认，进入下一阶段

**动态更新知识图谱：** 每阶段完成后，补充新发现的概念和关系到 `knowledge-graph.md`。

## 进度管理

`progress.json` 结构：
```json
{
  "topic": "原始主题",
  "slug": "目录名",
  "current_stage": 2,
  "recommended_stages": 5,
  "stages": {
    "0": {"completed": true, "date": "2026-05-05"},
    "1": {"completed": true, "date": "2026-05-06"}
  }
}
```

---

## 防跑偏机制

**为什么需要：** 学习容易偏离规划，深入某个细节而忽略整体。

**机制：**
- 闸门处的 `F: 回归路线图` — 重新读取 roadmap.md，检查对齐
- 阶段 1 完成后提醒确认路线图
- 知识图谱持续更新，让你看到学习轨迹

---

## 完成

达到推荐阶段数后，展示摘要：
```
━━━ 学习完成！━━━
主题: [topic]
完成阶段: [N]/[recommended_stages]

📁 输出：
  roadmap.md、overview.md、knowledge-graph.md
  stage-1~N/（notes、quiz、practice）

💡 建议：回顾 overview.md 和 knowledge-graph.md 巩固结构
```

---

## 注意事项

- **科普语言通俗：** 避免术语堆砌，用类比和案例
- **知识图谱动态更新：** 每阶段补充新发现
- **阶段数自适应：** 根据主题复杂度推荐 3-7 阶段
- **详细模板位置：** `references/templates.md`
- **阶段指导位置：** `references/learning-phases.md`
