# Self-Learning — 通用自学系统

## 触发条件

当用户输入以 `/learn` 开头时激活本技能。

----

## 命令与路由

| 命令 | 行为 |
|------|------|
| `/learn <topic>` | 启动新学习任务：生成 slug，创建 `<slug>/` 目录，进入阶段 0 |
| `/learn` | 恢复最近一个未完成的学习任务 |
| `/learn status` | 显示当前学习进度面板 |
| `/learn stop` | 中途退出当前学习任务，保存已完成的部分 |
| `/learn stages <N>` | 修改剩余阶段数（如 `/learn stages 3` 表示只学到阶段 3） |

----

## 核心流程

### 启动新任务 (`/learn <topic>`)

1. **生成 slug：**
   - 转小写
   - 空格替换为 `-`
   - 移除非字母数字、破折号、下划线之外的字符
   - 合并连续破折号为单个
   - 去除首尾破折号
   - 若结果为空字符串，使用 `untitled`

2. **创建目录结构：**
   ```
   <slug>/
   ├── roadmap.md           # 学习路线图
   ├── overview.md          # 科普性整体介绍（故事化串联）
   ├── knowledge-graph.md   # 全局知识图谱（结构化展示）
   ├── progress.json        # 学习进度
   ├── stage-1-foundations/
   ├── stage-2-principles/
   ├── stage-3-advanced/
   ├── stage-4-practice/
   ├── stage-5-synthesis/
   └── stage-6-frontier/
   ```

3. **初始化 `progress.json`：**
   ```json
   {
     "topic": "<原始输入>",
     "slug": "<slug>",
     "current_stage": 0,
     "stages": {}
   }
   ```

4. **进入阶段 0（路线图、科普介绍与知识图谱）：**
   - 搜索并生成学习路线图、科普性整体介绍和全局知识图谱
   - 建立全局认知框架后，闸门确认进入阶段 1

### 恢复任务 (`/learn`)

1. 扫描 `*/progress.json`，找到存在未完成阶段的任务（判定条件：`current_stage < recommended_stages` 且 `stages[current_stage].completed` 为 `false`）
2. 若有多个，列出供用户选择，同时显示各任务的推荐阶段数和当前进度
3. 若仅一个，自动恢复，显示提示：`恢复学习 [topic]，当前阶段 N/[recommended_stages]`
4. 若没有，提示：`没有进行中的学习任务，请使用 /learn <topic> 开始`

### 中途退出 (`/learn stop`)

1. 读取当前任务的 `progress.json`
2. 显示已完成阶段摘要
3. 保存当前进度，更新 `recommended_stages` 为 `current_stage`（表示只学到当前阶段）
4. 提示：`已保存学习进度，完成 [N] 个阶段。可随时使用 /learn 恢复继续学习。`

### 修改阶段数 (`/learn stages <N>`)

1. 读取当前任务的 `progress.json`
2. 更新 `recommended_stages` 为 `<N>`
3. 若 `current_stage >= <N>`，提示：`当前已完成阶段 [current_stage]，超出目标阶段数 <N>。学习任务已完成。`
4. 否则，提示：`已调整剩余阶段数为 <N>，继续学习剩余 [N - current_stage] 个阶段。`

### 查看状态 (`/learn status`)

显示进度面板：

```
━━━ 学习进度 ━━━
主题: [topic]
当前阶段: 阶段 N — [阶段名称]
已完成: [X/7] 阶段
━━━━━━━━━━━━━━━
```

## 阶段 0：路线图、科普介绍与知识图谱生成

阶段 0 是特殊的起始阶段，为后续所有阶段奠定规划基础。**核心目标：先建立全局认知，再深入细节学习。**

### 设计理念

传统学习路径往往直接进入各阶段学习，导致学习者只见树木不见森林。本技能采用"全景式学习法"：

```
科普介绍（感性认知）→ 知识图谱（理性框架）→ 各阶段深入学习（逐层展开）→ 知识整合（融会贯通）
```

**阶段 0 的三步产出：**
1. **路线图 (`roadmap.md`)**：规划性的阶段安排，告诉"学什么、学多久"
2. **科普介绍 (`overview.md`)**：故事性的知识串联，告诉"这个领域是怎么回事，各知识点如何自然衔接"
3. **知识图谱 (`knowledge-graph.md`)**：结构性的知识网络，告诉"知识点如何依赖、如何关联"

**科普介绍与知识图谱的关系：**
- **科普介绍**：用通俗语言讲故事，像导游带游客浏览景点，建立感性认知
- **知识图谱**：用结构化方式展示骨架，像地图标注路线，建立理性框架
- 两者配合：感性认知 + 理性框架 = 完整的全局认知

### 流程

1. **研究主题领域：**
   - 搜索该领域的标准学习路径（如大学课程大纲、官方文档推荐学习路径、社区公认学习路线）
   - 识别核心子领域和依赖关系
   - 确定各阶段边界
   - **梳理知识节点及其依赖关系**（为知识图谱做准备）
   - **收集领域的经典案例、历史发展、应用场景**（为科普介绍做准备）

   4. **遇到困难：** 检查图谱中的依赖概念是否掌握

   ## 七、动态更新

   > 知识图谱会在每个阶段完成后动态更新，补充该阶段发现的新的知识节点和关系。
   ```

5. **闸门展示：**

   💡 学习建议：
      1. 先读 overview.md（感性认知，10-15 分钟）

6. **等待用户输入：**
   - `O` — 展示科普介绍的详细内容（故事线、知识衔接、常见问题等），重新展示闸门
   - `V` — 展示知识图谱的详细内容（概念索引、学习路径可视化等），重新展示闸门
   - `Q: <question>` — 回答疑问后重新展示闸门
   - `F: <feedback>` — 根据反馈调整路线图、科普介绍和知识图谱后重新展示闸门
   - `N` — 确认路线图、科普介绍和知识图谱，保存 `progress.json`，进入阶段 1

7. **闸门通过后：**
    
    a. **评估主题复杂度：**
       - **简单主题**（如具体技术点、单一工具）：范围明确、依赖少、可快速掌握 → 推荐 3-4 阶段
       - **中等主题**（如框架学习、子领域）：有多个子主题、需要系统性理解 → 推荐 5-6 阶段
       - **复杂主题**（如整个领域、跨学科主题）：范围广、依赖多、需要长期学习 → 推荐 7 阶段
    
    b. **展示阶段数建议：**

    c. **等待用户选择：**
       - `A` — 按推荐阶段数执行，跳过剩余阶段
       - `M: <number>` — 用户自定义阶段数（如 "M: 4" 表示只学 4 个阶段）
       - `N` — 使用完整 7 阶段流程
    
    d. **保存选择到 progress.json：**
       ```json

    e. **提示用户：**
       `路线图已生成，根据主题复杂度，推荐学习 [N] 个阶段。后续阶段将按此展开，当内容偏离时可随时使用 F: 回归路线图。在学习过程中，可随时使用 /learn stop 中途退出。`

1. 官方文档 / 官方网站（必须包含至少 1 个，标注为 "官方权威"）
2. 权威教材 / 经典书籍（标注为 "教材权威"）
3. Google Scholar / 学术论文（标注为 "学术权威"）
4. Wikipedia（标注为 "百科参考"）
5. Reddit（相关 subreddit）
6. Stack Overflow / Stack Exchange
7. 博客 / 教程（标注为 "社区实践"）

- 每个阶段至少引用 1 个官方权威资源

- [ ] 是否包含实际应用场景的题目（而非纯术语定义）
- [ ] 是否包含需要推理分析的题目
- [ ] 是否有完整的答案解析

**实战任务质量检查：**
- [ ] 是否有明确的可执行步骤（不少于 3 步）

3. 如有偏离，调整笔记/测验/实战
4. 重新展示闸门

## 注意事项

- 所有文件使用 Markdown 格式
- 日期格式统一为 `YYYY-MM-DD`
- Stage 子目录名称为固定值，不可更改
- `progress.json` 必须保持有效的 JSON 格式
- 每个阶段生成笔记、测验、实战三个文件，缺一不可
- 用户可用 `Q` / `F` 在闸门处反复交互，直到满意才 `N` 进入下一阶段
- **科普介绍与知识图谱配合使用**：科普介绍建立感性认知，知识图谱建立理性框架
- **科普介绍使用通俗语言**：避免专业术语堆砌，用类比和案例帮助理解
- **知识图谱是动态更新的**：每阶段完成后会补充新发现的概念和关系
- **知识图谱使用 Mermaid 语法**：确保渲染兼容性
- **阶段 0 的三份文档是学习的关键**：建议学习者在进入各阶段前仔细阅读 overview.md 和 knowledge-graph.md
