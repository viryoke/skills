# 学习阶段参考

本文档定义了由 `self-learning` 技能支持的自主学习框架中的 7 个学习阶段（阶段 0-6）。每个阶段都配有目标、研究指引、笔记重点、测验方向和实战方向。

---

## Stage 0: 调研与路线图

**目标:** 快速了解领域全貌，制定系统性学习路线。

**产出:** `roadmap.md`

**研究指引:**

- 搜索该领域的概述、入门指南、公认的学习路径
- **优先搜索官方资源：**
  - `[topic] official documentation`
  - `[topic] official website guide`
  - `[topic] specification`（如适用）
- 识别领域的核心分支和子领域
- 确定学习顺序的依赖关系（哪些必须先学）
- **主题复杂度评估标准：**
  - **简单主题标志：** 具体技术点、单一工具、范围明确（如 "React useEffect"、"Docker 基础命令"）
  - **中等主题标志：** 框架学习、子领域、有多个核心概念（如 "React Hooks"、"机器学习基础"）
  - **复杂主题标志：** 整个领域、跨学科主题、范围广（如 "前端开发"、"人工智能"）

**笔记重点:**

- 领域定义与范围的清晰表述
- 核心分支与子领域清单
- 学习依赖关系图
- 推荐资源列表及其评价

**测验方向:**

- 能否清晰描述该领域的定义和范围？
- 能否列举核心分支并说明各分支之间的关系？
- 能否解释为什么某些主题必须先学？

**实战方向:**

- 生成 `roadmap.md` 文档
- 收集并整理推荐资源
- 制定个人学习时间表

## Stage 1: 基础概念

**目标:** 建立核心术语体系，掌握入门知识框架。

**研究指引:**

- 搜索 `[topic] beginner guide`, `[topic] introduction`, `[topic] fundamentals`
- **强制包含官方文档：**
  - `[topic] official documentation getting started`
  - `[topic] official tutorial`
- 关注：核心定义、基本术语、入门级概念关系
- 优先找权威教材/教程的目录结构作为组织框架
- **笔记质量要点：**
  - 必须包含至少 3 个具体示例（代码片段、配置示例、命令示例）
  - 示例必须可执行或有明确说明
  - 核心术语必须标注来源（官方权威/教材权威）

**笔记重点:**

- 核心术语及其定义表格
- 基本概念之间的关系图
- 入门级知识体系全景

**测验方向:**

- 术语定义的识记
- 基本概念的辨析
- 简单场景的判断

**实战方向:**

- 搭建学习环境
- 完成入门级示例/Hello World
- 手动验证基本概念

---

## Stage 2: 核心原理

**目标:** 深入理解工作原理和底层机制。

**研究指引:**

- 搜索 `[topic] how it works`, `[topic] internals`, `[topic] principles`
- **强制包含官方文档深入部分：**
  - `[topic] official documentation deep dive`
  - `[topic] architecture documentation`
  - `[topic] implementation details`
- 关注：运行机制、数据流向、算法原理、设计思想
- 用类比和图示帮助理解抽象概念
- **笔记质量要点：**
  - 必须包含 UML 图或架构图（PlantUML/Graphviz）
  - 图示必须有完整注释说明
  - 原理解释必须有具体实例支撑

**笔记重点:**

- 工作原理的逐步解析
- UML 图解（PlantUML: 类图、时序图、流程图等）
- 与其他相关概念的对比

**测验方向:**

- 原理机制的推理分析
- 给定场景下的行为预测
- 概念对比与区分

**实战方向:**

- 手动模拟或复现核心机制
- 阅读并注释关键源码/文献
- 编写 mini 实现验证理解

---

## Stage 3: 进阶专题

**目标:** 掌握高级主题和专项领域知识。

**研究指引:**

- 搜索 `[topic] advanced`, `[topic] best practices`, `[subtopic] deep dive`
- **强制包含官方最佳实践：**
  - `[topic] official best practices`
  - `[topic] official performance guide`
  - `[topic] official security guide`
- 关注：高级用法、最佳实践、性能优化、安全考量、专项子领域
- 收集不同专家的观点进行对比
- **笔记质量要点：**
  - 必须包含对比表格（不同方案/观点）
  - 高级用法必须有完整示例（包括错误用法示例）
  - 最佳实践必须有实际应用场景

**笔记重点:**

- 进阶概念与技术详解
- 架构图（PlantUML + Graphviz 绘制）
- 不同方案/观点的对比表格

**测验方向:**

- 复杂场景的最优方案选择
- 多方案对比分析
- 陷阱识别与规避

**实战方向:**

- 实现进阶功能/特性
- 优化已有实现
- 处理边界情况和异常场景

---

## Stage 4: 实战项目

**目标:** 通过完整项目将知识转化为实践能力。

**研究指引:**

- 搜索 `[topic] project tutorial`, `[topic] real world example`, `[topic] case study`
- **强制包含官方示例项目：**
  - `[topic] official example project`
  - `[topic] official starter template`
  - `[topic] official deployment guide`
- 关注：项目架构设计、开发流程、测试部署
- 收集工业界实际案例
- **笔记质量要点：**
  - 必须包含完整的项目架构图（PlantUML/Graphviz）
  - 技术决策必须有明确的理由和权衡分析
  - 踩坑记录必须有解决方案代码

**笔记重点:**

- 项目架构设计（PlantUML + Graphviz 架构图）
- 关键技术决策及理由
- 踩坑记录与解决方案

**测验方向:**

- 项目设计决策分析
- 给定需求的设计方案
- 代码审查与改进

**实战方向:**

- 从零搭建一个完整项目
- 实现核心功能模块
- 编写测试用例
- （如适用）部署上线

---

## Stage 5: 融会贯通

**目标:** 串联所有知识，形成系统性理解框架。

**研究指引:**

- 搜索 `[topic] comprehensive review`, `[topic] survey`, `[topic] comparison`
- **强制包含权威综述：**
  - `[topic] academic survey paper`
  - `[topic] official overview`
  - `[topic] state of the art`
- 关注：知识体系全景图、不同流派的对比、历史演进脉络
- 阅读综述性论文/文章
- **笔记质量要点：**
  - 必须包含完整的知识图谱（Mermaid 思维导图）
  - 历史演进必须有时间线图
  - 知识关联必须有清晰的逻辑链条

**笔记重点:**

- 全景知识图谱（Mermaid 思维导图）
- 知识间关联关系分析
- 历史演进与未来趋势
- 个人知识体系的总结

**测验方向:**

- 跨领域知识的综合分析
- 前沿问题的开放性讨论
- 系统性问题的诊断与解决

**实战方向:**

- 撰写该领域的综述/总结文章
- 给他人讲解该领域知识（费曼技巧）
- 构建个人知识库体系

---

## Stage 6: 跟踪前沿

**目标:** 了解最新动态，建立持续学习的信息渠道。

**研究指引:**

- 搜索 `[topic] latest developments`, `[topic] 2025/2026`, `[topic] research papers`
- 搜索 Reddit、Hacker News、Twitter/X 上的最新讨论
- **强制包含最新官方动态：**
  - `[topic] official blog` / `[topic] official changelog`
  - `[topic] official roadmap`
  - `[topic] latest release notes`
- 关注：最新论文、技术突破、行业趋势、重要会议/活动
- **笔记质量要点：**
  - 最新进展必须有明确的时间标注（年份/月份）
  - 重要论文必须有关键贡献摘要（不是完整复述）
  - 行业趋势必须有数据支撑或权威来源引用

**笔记重点:**

- 最新进展摘要
- 重要论文的关键贡献
- 行业趋势分析

**测验方向:**

- 新技术的应用场景判断
- 行业趋势的预测分析
- 技术选择的权衡判断

**实战方向:**

- 复现最新论文中的实验/方法
- 试用最新的工具/框架
- 规划个人的持续学习路径

**社区与信息渠道:**

- **学术渠道:** Google Scholar, arXiv, ACM, IEEE
- **社区渠道:** Reddit (r/\<topic\>), Stack Exchange, Discord/Slack 社区
- **新闻渠道:** Hacker News, TechCrunch, 行业周刊/Newsletter
- **资源渠道:** GitHub Trending, Papers With Code, 顶会论文列表
